# Paper Wiki — 打造你专属的科研知识库

> 每天面对成百上千篇新论文，不知道哪些值得读？读完之后笔记散落在各处，想回顾却找不到？Paper Wiki 帮你建立一套完整的个人科研知识库，让发现、阅读、整理、回顾形成闭环。

---

## 这是什么？

Paper Wiki 是一个**科研知识库管理工具**，它把论文的"发现 → 分析 → 存储 → 检索"整条链路串在一起，用 [Obsidian](https://obsidian.md/) 作为存储后端，让你拥有完全属于自己的、可长期积累的论文知识库。

你可以把它理解为：
- **一个自动化的论文助手** —— 每天帮你从 arXiv 和顶会中筛选出最值得读的论文
- **一个结构化的笔记系统** —— 每篇论文都有统一的深度分析模板，图文并茂
- **一个可检索的知识库** —— 所有笔记通过标签和双向链接串联，随时搜索回顾
- **一个持续积累的研究档案** —— 读过的论文、做过的分析永远不会丢失

---

## 它能做什么？

### 1. 每日论文推荐 `/paper-wiki daily`

每天早上，自动从 arXiv、Semantic Scholar 等来源搜索最新论文，根据你设定的研究方向和关键词进行筛选，按**相关性、新近性、热门度、质量**四个维度评分，生成当天的推荐列表。

- 自动过滤你不感兴趣的领域
- 自动排除已分析过的论文
- 前3篇高分论文自动生成深度分析 + 提取配图
- 所有推荐笔记保存在 `10_Daily/` 目录，按日期归档

### 2. 单篇深度分析 `/paper-wiki analyze <论文ID>`

对任意一篇论文进行深度分析，生成结构化的笔记：

- 摘要翻译 + 核心要点提炼
- 研究背景与动机
- 方法概述（核心思想、框架、各模块详解、数学公式）
- 实验结果（数据集、基线、主要结果、消融实验）
- 深度分析（研究价值、优势、局限性、适用场景）
- 与相关论文对比（3篇对比，含表格）
- 技术路线定位 + 未来工作建议
- 综合评价（0-10 分制，五个维度）

同时自动下载 PDF 到知识库，提取论文中的配图，方便你在 Obsidian 中直接查看。

### 3. 顶会论文追踪 `/paper-wiki conf <会议> <年份>`

追踪 CVPR、NeurIPS、ICML、ICLR、CCS 等顶级会议的最新论文：

- 从 DBLP 获取完整会议论文列表
- 用你的关键词自动筛选相关论文
- Semantic Scholar 补充引用和摘要信息
- 生成会议专题推荐报告

### 4. 笔记快速搜索 `/paper-wiki search <关键词>`

在已整理的所有笔记中快速搜索：

- 支持标题、作者、关键词、标签搜索
- 自动按相关性排序
- 支持组合查询

### 5. 知识库维护 `/paper-wiki sync`

定期维护知识库的健康状态：

- 重建全局索引页
- 检测失效的内部链接
- 发现孤立页面（无入链的笔记）
- 更新来源追踪清单

## 目录结构

所有数据存储在固定的 `D:\paper-wiki` 目录下：

```
D:\paper-wiki\
├── _sources\papers\      # 下载的论文 PDF
├── references\           # 论文分析报告（你的核心资产）
├── 10_Daily\            # 每日推荐笔记
├── concepts\            # 核心概念（如后门攻击、联邦学习）
├── entities\            # 实体（人物、会议、工具）
├── synthesis\           # 综合与综述
├── journal\             # 研究日志
├── index.md             # 全局索引
└── log.md               # 操作日志
```

**设计原则**：
- 所有 PDF 统一归档，永远不会丢失
- 所有分析报告统一存放，随时检索
- Obsidian 的双向链接和图谱视图，让知识自然生长

## 快速开始

### 第一步：初始化你的知识库

输入：
```
/paper-wiki
```

系统会引导你完成初始化：
1. **输入研究方向** —— 例如"联邦学习后门攻击"
2. **系统自动生成关键词** —— 基于方向名称推断英文核心术语
3. **确认或修改关键词** —— 你可以增删改，确保精准匹配
4. **确认创建** —— 一键生成完整的知识库目录和配置文件

### 第二步：获取今日推荐

```
/paper-wiki daily
```

等待片刻，系统会搜索最新论文并生成推荐报告。前3篇论文会自动提取图片并生成详细分析。

### 第三步：深度分析感兴趣的论文

```
/paper-wiki analyze 2402.12345
```

输入 arXiv ID，系统会自动：
1. 下载 PDF 到知识库
2. 提取论文配图
3. 生成完整的结构化分析报告到 `references/`

### 第四步：随时搜索回顾

```
/paper-wiki search backdoor attack
```

在所有已整理的笔记中快速搜索，找到你需要的内容。

## 配置说明

所有配置集中在 `D:\paper-wiki\paper-wiki-config.yaml`：

```yaml
research_direction:
  name: "你的研究方向"
  keywords:
    - "关键词1"
    - "关键词2"
  excluded_keywords:
    - "survey"

search:
  arxiv:
    categories: ["cs.LG", "cs.AI", "cs.CR"]
  dblp:
    conferences: ["NeurIPS", "ICML", "ICLR"]

scoring:
  daily:
    relevance: 0.40   # 相关性权重
    recency: 0.20     # 新近性权重
    popularity: 0.30  # 热门度权重
    quality: 0.10     # 质量权重
```

修改配置后，下次运行命令时会自动生效。

## 为什么用 Obsidian？

- **双向链接**：`[[论文标题]]` 让笔记之间自然关联
- **图谱视图**：直观看到论文、概念、作者之间的关系网络
- **本地存储**：所有数据都在你的硬盘上，完全私有
- **Markdown 格式**：通用格式，随时迁移
- **丰富的插件生态**：可以安装各种插件扩展功能

---

## 使用建议

1. **每日浏览推荐**：养成每天查看 `10_Daily/` 的习惯，快速了解领域动态
2. **深度分析要读的论文**：不要每篇都精读，只分析真正感兴趣的
3. **善用标签**：分析报告中的 tags 会成为你检索的重要入口
4. **定期维护**：每隔一段时间运行 `/paper-wiki sync`，保持知识库整洁
5. **积累概念页**：在 `concepts/` 中维护核心概念的笔记，形成领域知识框架
